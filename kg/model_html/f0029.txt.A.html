[0] Cross-match spatially clusters and organizes several astronomical point-source measurements from one or more surveys. [1] Ideally, each object would be found in each survey. [2] Unfortunately, the observation conditions and the objects themselves change continually. [3] Even some stationary objects are missing in some observations; [4] sometimes objects have a variable light flux and sometimes the seeing is worse. [5] In most cases we are faced with a substantial number of differences in object detections between surveys and between observations taken at different times within the same survey or instrument. [6] Dealing with such missing observations is a difficult problem. [7] The first step is to classify misses as ephemeral -- when the object moved or simply disappeared, masked -- when noise hid or corrupted the object observation, or edge -- when the object was near the edge of the observational field. [8] This classification and a spatial library to represent and manipulate observational footprints help construct a Match table recording both hits and misses. [9] Transitive closure clusters friends-of-friends into object bundles. [10] The bundle summary statistics are recorded in a Bundle table. [11] This design is an evolution of the Sloan Digital Sky Survey cross-match design that compared overlapping observations taken at different times.
