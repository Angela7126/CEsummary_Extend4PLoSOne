[0] This paper reexamines the rules of thumb for the design of data storage systems. [1] Briefly, it looks at storage, processing, and networking costs, ratios, and trends with a particular focus on performance and price/performance. [2] Amdahl's ratio laws for system design need only slight revision after 35 years -- the major change being the increased use of RAM. [3] An analysis also indicates storage should be used to cache both database and web data to save disk bandwidth, network bandwidth, and people's time. [4] Surprisingly, the 5-minute rule for disk caching becomes a cache -- everything rule for web caching.
