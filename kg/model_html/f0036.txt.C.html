[0] This paper reexamines the rules of thumb for the design of data storage systems. [1] Briefly, it looks at storage, processing, and networking costs, ratios, and trends with a particular focus on performance and price/performance. [2] Amdahl's ratio laws for system design need only slight revision after 35 years -- the major change being the increased use of RAM. [3] An analysis also indicates storage should be used to cache both database and web data to save disk bandwidth, network bandwidth, and people's time. [4] Surprisingly, the 5-minute rule for disk caching becomes a cache -- everything rule for web caching. [5] Data stores will become huge. [6] Our biggest challenge is to make it easy to access and manage them. [7] Automating all the tasks of data organization, accesses, and protection. [8] Disk technology is overtaking tapes, but at the same time disks are morphing into tape-like devices with primarily sequential access to optimize the use of disk arms. [9] Meanwhile, RAM improvements encourage us to build machines with massive main memory. [10] Indeed, the main change to Amdahl's balanced system law is that alpha (=MIPS/RAM size) is rising from 1 to 10. [11] Network bandwidth is improving at a rate that challenges many of our design assumptions. [12] LAN/SAN software is being streamlined so it is no longer the bottleneck. [13] This may well allow a re -centralization of computing. [14] Still, data caching is an important optimization. [15] Disk caching still follows the 5-minute random rule and the one-minute sequential rule. [16] Web caching encourages designs that simply cache all pages.
