[0] As mentioned earlier, scientists often start with numeric data arrays from their instruments or simulations. [1] Often, these arrays are accompanied by tabular data describing the experimental setup, simulation parameters, or environmental conditions. [2] These arrays eventually undergo feature extraction to produce tabular objects. [3] Most scientific studies involve exploring and data mining these object-oriented tabular datasets. [4] However, traditional procedural tools always search for relevant files and then open each file to extract all relevant data. [5] So they runs slower and slower as data volumes increase. [6] However, set-oriented file processing will perform analysis to "all data with these attributes" rather than working on a list of file/directory names. [7] The database community has provided automatic query processing along with CPU and IO parallelism for over two decades. [8] We believe that many of those techniques apply to scientific datasets .
