[0] Database systems usually do not support science's core data type: the N-dimensional array. [1] This collection is generally called the impedance mismatch, i.e, the mismatch between the programming model and the database capabilities, thus it is difficult to map many science applications into conventional tabular database systems. [2] However, current databases are being integrated with programming languages so that they can support object-oriented databases. [3] This new generation of object relational database systems treats any data type as an encapsulated type that can be stored as a value in a field of a record. [4] This linking-embedding object model also nicely accommodates the integration of database systems and file systems -- files are treated as linked-objects. [5] We believe this database, file system, and programming language integration will be the key to managing and accessing peta-scale data management systems in the future.
