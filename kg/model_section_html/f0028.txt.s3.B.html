[0] These peta-scale datasets required a new work style. [1] Science data centers that provide access to both the data and the applications that analyze the data are emerging as service stations for one or another scientific domain. [2] The new work style in these scientific domains is to send questions to applications running at a data center and get back answers, rather than to bulk-copy raw data from the archive to your local server for further analysis. [3] This minimizes data movement and allows collaboration among a group of scientists doing joint analysis. [4] When a scientist wants to correlate data from two different data centers, then there is no option but to move part of the data from one place to another. [5] If this is common, the two data centers will likely federate with one another to provide mutual data backup since the data traffic will justify making the copy. [6] Such systems have to have a mechanism in place to protect against data loss, and provide availability even with a less than full configuration -- a self-healing system is required. [7] Geographic replication provides both data availability and protects against data loss. [8] All these scenarios postulate easy data access, interchange and integration. [9] Data must be self-describing in order to allow this.
