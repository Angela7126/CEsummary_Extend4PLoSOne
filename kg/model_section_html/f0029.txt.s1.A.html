[0] Given several observations of the sky, called runs, astronomers often want to cross-match all the observations of each object from all runs that observed that object. [1] Two objects are said to match if they come from different runs and if their positions differ by less than their classification distance. [2] Picking the classification distance depends on the data and on the intended use of the cross-match. [3] If only stationary objects are to be matched, then the classification distance can be a small multiple of the maximum of the two object's circular rms position errors. [4] However, when comparing data from different instruments or from times with different seeing, the position uncertainty, astrometric precision, observational errors or chance overlays added uncertainty. [5] We want a classification distance that minimizes the overall error. [6] First, let us focus on cross-match stationary objects. [7] Given two runs that overlap, if object O1 observed in run1 matches object O2 in run2, we call the pair an O1-rn2 hit. [8] If there are no O1-run2 hits, we call it an O1-run2 miss. [9] O1-run2 misses can have three generic causes (see Figure 1): [10] Ephemeral: [11] O1 is beyond the detection threshold or the seeing is bad in run2 or O1 may be invisible if run2 is a different kind of instrument, or O1 is a variable or transient object which varied below the detection threshold in run2, or O1 moved more than the classification distance between the two observations. [12] Masked: [13] O1 was fully masked by something like a meteor trail, satellite or refraction of a bright object in run 2. [14] Edge: [15] O1 was on the edge of the run2 footprint and so not all its pixels were observed. [16] Our goal is to compute a table Match( run1, objectID1, run2, objectID2, hitOrMiss), where hitOrMiss takes on one of the values Hit, Ephemeral, Masked, or Edge. [17] When the objectID1-run2 pair is a miss, then objectID2 is zero, and we need to compute whether hitOrMiss equals to Ephemeral, Masked, or Edge.
