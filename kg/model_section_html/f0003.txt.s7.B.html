[0] SLN consists of two facets: form and semantics. [1] The basic form of Semantic Link Network is SLN=. [2] (1) N is a set of semantic nodes. [3] Semantic nodes are usually explicit. [4] (2) L is a set of semantic links. [5] Semantic links are usually implicit and rely on semantic nodes' attributes and behaviors. [6] (3) Rules is a set of rules such that new semantic links can be appropriately added or the implicit semantic links can be found between semantic nodes as the effect of interaction, reasoning, influence, or evolution. [7] Two basic types of operations are Add and Delete, which can be specialized into the following six basic operations: [8] AddLink, DelLink, AddNode, DelNode, AddRule, and DelRule. [9] SLN is an abstraction of various autonomous relational networks in different spaces. [10] It can be specialized for dynamic or functional modeling by incorporating the class mechanisms of the object-oriented method. [11] Fig. 8 depicts the way to enrich SLN for detailed function modeling by making use of OOMD, OOP (Object-Oriented Programming), and SOA (Service-Oriented Architecture), in the background of interactions between multiple spaces. [12] Semantic images emerge with reflection, linking, reasoning and socio interactions as depicted in Fig. 9. [13] All possible semantic images constitute a semantic space, which can be modeled by integrating multi-dimensional classifications, the semantic links between points and between coordinates, the rules of linking and constraints, and relevant operations [89]. [14] Mapping SLN into the semantic space concerns the following mappings: [15] (1) Semantic link mapping [16] (2) Semantic node mapping [17] (3) Community mapping [18] (4) SLN mapping [19] (5) Mapping an SLN into a semantic range [20] SLN is not just a labeled graph, it has the following characteristics: [21] (1) Openness. [22] (2) Dynamicity. [23] (3) Self-organization. [24] (4) Autonomous reasoning. [25] (5) Operation order sensitive. [26] (6) Complex. [27] (7) Semantic image. [28] (8) Evolving community. [29] SLN research concerns the following issues: [30] (1) How to efficiently answer the query on the relations between seemly irrelevant objects or events? [31] (2) How to recommend the best link to a node to realize its maximum performance according to such criteria as the rise of rank. [32] (3) How to develop SLN toward a self-organized semantic model to support current network applications? How to support efficient relational query, especially, query on the evolving relations? [33] (4) How to predict the evolution of SLN? How do semantic links and semantic nodes influence each other during evolution? [34] (5) How to enable an SLN to support automatic question answering and explanation? How can a query be efficiently routed in a large self-organized semantic link network through explicit or implicit semantic link paths? [35] (6) How do different communities and reasoning processes influence each other over a large SLN? [36] (7) What level of intelligence can be reached by integrating multi-dimensional classification ability, linking ability and complex reasoning ability? [37] The above discussion shows that SLN is different from previous links and graph-based models in goal, model, method and research issues.
