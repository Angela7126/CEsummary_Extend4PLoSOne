[0] Consider the problem of designing a machine to solve well-defined intellectual problems. [1] We call a problem well-defined if there is a test which can be applied to a proposed solution. [2] Since any test may be regarded as being performed by a Turing machine, this means that welldefined intellectual problems may be regarded as those of inverting functions and partial functions defined by Turing machines. [3] This paper deals with the problem of designing a Turing machine which, when confronted by the number pair (m,r), computes as efficiently as possible a function g(m,r) such that fm(g(m,r)) = r. [4] Essentially, it substitutes integers in fm(n) until it comes to one such that fm(n) = r. [5] It is not defined for a given value of n if the computation does not come to an end. [6] Hence the machine calculates the numbers fmk(n) in some order where fmk(n) is fm(n) if the computation of fm(n) ends after k steps and is otherwise undefined. [7] Our problem does not end once we have found this procedure for computing g(m,r) because this procedure is extremely inefficient.
