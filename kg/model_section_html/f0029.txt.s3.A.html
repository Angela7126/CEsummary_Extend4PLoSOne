[0] Computing misses is more complex. [1] First, we need to know for each object O1 in run1 what other runs overlap O1 to within the run-run2 classification distance. [2] In order to do so, we create a Overlap table and filled it with all Overlap records find in run1 and run 2. [3] For each Overlap record, we uses the HTM fRegionGetObjectsFromRegionId function to search the catalog for run1 objects that are in the run1-run2 overlap region but do not yet have a run2 entry in the Match table. [4] When this is done, the Miss table lists all the O1-Run2 misses. [5] Next, we need to judge if the missing object O1 is either near the run2 footprint edge or is inside a run2 mask. [6] We categorize each miss as ephemeral, masked or edge, and put that characterization in the Match table. [7] First, we find the edge cases by using the Overlap.OverlapRegionEdgeID. [8] Then, we use the Overlap.run2MasksID which is region ID of the union of the run2 and the HTM code to identify all the Miss objects inside the mask region. [9] The residual misses must be ephemeral.
