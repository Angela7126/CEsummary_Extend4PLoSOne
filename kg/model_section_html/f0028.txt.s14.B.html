[0] Science centers that curate and serve science data are emerging around next-generation science instruments. [1] Because data collection is now separated from data analysis, extensive metadata describing the data in standard terms is needed so people and programs can understand the data. [2] There is a convergence of the nascent-databases (HDF, NetCDF, FITS, ...) which focus primarily on the metadata issues and data interchange, and the traditional data management systems (SQL and others) that have focused on managing and analyzing very large datasets. [3] We believe the emphasis on extending database systems by unifying databases with programming languages so that one can either embed or link new object types into the data management system will enable this synthesis. [4] Three technical advances will be crucial to scientific analysis: [5] (1) extensive metadata and metadata standards; [6] (2) great analysis tools; [7] and (3) set-oriented data parallelism access supported by new indexing schemes and new algorithms. [8] Science centers will provide both the archives and the institutional infrastructure to develop these peta-scale archives and the algorithms and tools to analyze them.
