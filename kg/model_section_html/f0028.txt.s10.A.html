[0] File systems have no metadata beyond a hierarchical directory structure and file names. [1] They encourage a do-it-yourself data-model and access-model, lack a high-level query language, and cannot manage billions of files. [2] However, we see NetCDF, HDF, FITS, and Google Map-Reduce as nascent database systems rather than file systems. [3] This is because they have metadata, few indexing strategies, a simple data manipulation language, and non-procedure and parallel programming.
