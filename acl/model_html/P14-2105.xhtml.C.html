[0] We develop a semantic parsing framework based on semantic similarity for open domain question answering (QA. [1] We focus on single-relation questions and decompose each question into an entity mention and a relation pattern. [2] Using convolutional neural network models, we measure the similarity of entity mentions with entities in the knowledge base (KB) and the similarity of relation patterns and relations in the KB. [3] We score relational triples in the KB using these measures and select the top scoring relational triple to answer the question. [4] When evaluated on an open-domain QA task, our method achieves higher precision across different recall points compared to the previous approach, and can improve F 1 by 7 points. [5] In this work, we propose a semantic parsing framework for single-relation questions. [6] Compared to the existing work, our key insight is to match relation patterns and entity mentions using a semantic similarity function rather than lexical rules. [7] Our similarity model is trained using convolutional neural networks with letter-trigrams vectors. [8] This design helps the model go beyond bag-of-words representations and handles the OOV issue. [9] Our method achieves higher precision on the QA task than the previous work, Paralex , consistently at different recall points. [10] Despite the strong empirical performance, our system has room for improvement. [11] For instance, due to the variety of entity mentions in the real world, the parallel corpus derived from the WikiAnswers data and ReVerb KB may not contain enough data to train a robust entity linking model. [12] Replacing this component with a dedicated entity linking system could improve the performance and also reduce the number of pattern/mention candidates when processing each question. [13] In the future, we would like to extend our method to other more structured KBs, such as Freebase, and to explore approaches to extend our system to handle multi-relation questions.
